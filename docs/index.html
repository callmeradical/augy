Why augy

Everything a skill deserves

AI agents load skills from local directories. augy brings the discipline of a package manager to that chaos.

◆

SHA-based versioning

Every install tracks the exact GitHub commit SHA. Drift is detected the moment upstream changes.

⟳

Safe updates

Before overwriting, augy snapshots the current version. Roll back to any point in history.

⊞

Multi-agent

Deploy one skill to OpenCode, Claude, and Codex simultaneously. They update together.

⌕

Taps

Register trusted repos. Then augy install tdd — no URL needed.

≠

Diff browser

Interactive file-level diffs between any two versions. Know exactly what changed before upgrading.

⊙

Scan & import

Skills already on disk? Scan detects them, auto-discovers provenance, and brings them under management.

Get started

Up in three steps

1

Install

npm install -g @callmeradical/augy

Adds the augy command to your PATH.

2

Add a tap

augy tap add mattpocock/skills
3

Install a skill

augy install tdd

Or scan what you already have

If you've been installing skills manually, augy can find and import them:

augy scan    # finds untracked skills, detects provenance
Reference

Commands

All commands support --help for inline documentation.

install [url] [--agent <agents...>]

Install skills from a GitHub URL, shorthand, or bare name via taps. Shows a filterable multi-select when a repo contains multiple skills.

augy install tdd
augy install mattpocock/skills
augy install https://github.com/org/repo/tree/main
augy install tdd --agent opencode claude
scan

Walk all agent skill directories, find untracked skills, auto-detect provenance via git remotes and frontmatter, then import them into the registry.

augy scan
update [skill]

Check all installed skills for upstream SHA drift, show available upgrades, archive the current version, then deploy the new one.

augy update
augy update tdd
diff <skill> [sha1] [sha2]

Interactive file-level diff browser. Three modes: installed vs upstream, installed vs a specific SHA, or two local archives.

augy diff tdd
augy diff tdd abc1234
augy diff tdd abc1234 def5678
rollback <skill> [sha]

Restore a skill to a previously archived version. Without a SHA, presents an interactive picker of available snapshots.

augy rollback tdd
augy rollback tdd abc1234
list

Show all installed skills with SHAs, agents, update timestamps, and version history counts.

augy list
augy list --json   # raw registry JSON
info <skill>

Full metadata view: source, tap, SHA, install/update dates, per-agent paths with existence check, version history, and SKILL.md description preview.

augy info tdd
search [query]

Search all registered taps for available skills. Shows each skill's SHA and whether it's installed and up to date.

augy search
augy search tdd
tap add | remove | list

Manage trusted repos. Once tapped, skills can be installed by bare name. Validated on add by fetching the skill index.

augy tap add mattpocock/skills
augy tap add org/repo --path skills/engineering
augy tap list
augy tap remove mattpocock/skills
set-source <skill> <url>

Attach a GitHub URL to a skill imported without one. Accepts tree and blob URLs. Enables updates, diffs, and rollbacks going forward.

augy set-source commit https://github.com/org/repo/tree/main/skills/commit
uninstall <skill>

Remove a skill from all agent paths and the registry. Optionally prune archived versions from the version store.

augy uninstall tdd
pin / unpin <skill>

Pin a skill to freeze it at the current version, skipping it during augy update.

augy pin tdd
augy unpin tdd

Internals

The registry

augy stores all state in a single human-readable JSON lockfile at ~/.augy/registry.json. You can read, diff, and commit it just like any other config file.

Version archives are stored as plain directory snapshots — no binary formats, no databases.

~/.augy/ registry.json ← lockfile — source of truth for all installed skills versions/ tdd/ 7afa86d.../ ← snapshot before last upgrade abc1234.../ ← older snapshot commit/ def5678.../
